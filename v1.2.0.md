# KS Expire v1.2.0 — Plan de Desarrollo

## Resumen
Actualización mayor que transforma KS Expire de una app de seguimiento básica a una herramienta completa de control financiero personal. Incluye widgets, OCR, gráficas, categorías, modo oscuro profesional, reportes en PDF y detección inteligente de duplicados.

### FASE 6: OCR de recibos
**Estado:** ✅ Completada

**Objetivo:** Al tomar foto de un recibo, extraer automáticamente nombre del producto, precio, fecha de compra y fecha de vencimiento usando ML Kit OCR.

**Flujo:**
1. Usuario presiona "Tomar foto" en el formulario
2. Se abre la cámara (existente)
3. Al capturar, se procesa la imagen con ML Kit Text Recognition
4. Se extraen los campos detectados
5. Se muestra un diálogo de confirmación: "Detectamos estos datos: [nombre], [precio], [fecha]. ¿Son correctos?"
6. El usuario confirma o edita manualmente
7. Los campos del formulario se llenan automáticamente

**Detección inteligente:**
- **Nombre:** Buscar texto prominente (mayor tamaño de fuente), nombres de marcas conocidas
- **Precio:** Buscar patrones de moneda ($, €, ₡) seguidos de números con decimales
- **Fecha:** Buscar patrones de fecha (dd/mm/yyyy, mm-dd-yyyy, etc.)
- **Fallback:** Si no se detecta algo, dejar el campo vacío para llenado manual

**Limitaciones de cámara a manejar:**
- Imagen borrosa → Mostrar mensaje "La imagen parece borrosa, intenta de nuevo"
- Poca luz → Sugerir activar flash
- Sin texto detectado → "No se pudo leer texto en la imagen. Llena los datos manualmente."
- Texto parcial → Llenar solo lo que se detectó

**Dependencia:** com.google.mlkit:text-recognition:16.0.0 (offline, no requiere internet)

**Archivos a crear:**
- `OcrProcessor.kt` — Clase que procesa imagen y extrae datos
- `OcrResult.kt` — Data class con campos extraídos
- `OcrConfirmationDialog.kt` — Diálogo de confirmación de datos

**Archivos a modificar:**
- `CameraActivity.kt` — Pasar imagen a OCR después de captura
- `CameraViewModel.kt` — Estado de procesamiento OCR
- `CreateEditItemActivity.kt` — Recibir datos OCR y llenar formulario

### FASE 7: Detección de duplicados
**Estado:** ✅ Completada

**Objetivo:** Detectar y alertar cuando el usuario intenta crear un ítem que parece duplicado de uno existente.

**Criterios de detección:**
- Nombre similar (Levenshtein distance < 3, o contiene el mismo nombre base)
- Mismo tipo (suscripción/garantía)
- Precio similar (±10%)
- Ejemplo: "Netflix" y "Netflix Premium" → alerta

**Flujo:**
1. Al escribir el nombre en el formulario, buscar coincidencias en BD
2. Si hay coincidencia, mostrar banner amarillo debajo del campo nombre:
   "⚠️ Ya tienes un ítem similar: [nombre existente] ($X/mes). ¿Es el mismo?"
3. Botón "Ver existente" → abre el ítem existente
4. Botón "Es diferente" → cierra el banner y continúa

**Archivos a crear:**
- `DuplicateDetector.kt` — Lógica de detección

**Archivos a modificar:**
- `CreateEditItemViewModel.kt` — Verificar duplicados al cambiar nombre
- `CreateEditItemActivity.kt` — Mostrar banner de duplicado
- `activity_create_edit_item.xml` — Banner de duplicado
- `ItemDao.kt` — Query de búsqueda por nombre similar
- `ItemRepository.kt` — Método findSimilarItems()
